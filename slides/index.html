Key Contributions

1. Unified Multi-Service Orchestration

83 MCP-compliant tools across 15 services, accessible through natural language with per-user credential isolation.

2. RAG-Based Personalisation

Learns about you over time using locally-hosted embeddings (Ollama) and ChromaDB. Injects relevant memories per conversation.

3. Autonomous Vibe-Coding

15 workspace tools: clone, edit, grep, test, fix, commit, push, PR — with adaptive test-fix loops.

4. Self-Evolving Architecture

The agent can modify its own codebase. "Add a LinkedIn tool" → reads patterns, generates code, tests, PRs.

MCP Tool Architecture

Each service = self-contained FastMCP server, mounted into a composite registry with namespaces.

mcp = FastMCP("openpa")
mcp.mount(gmail_mcp,     namespace="gmail")      # gmail_send_email, gmail_get_unread, ...
mcp.mount(github_mcp,    namespace="github")     # github_create_pr, github_list_issues, ...
mcp.mount(workspace_mcp, namespace="ws")         # ws_workspace_run, ws_workspace_edit, ...
mcp.mount(spotify_mcp,   namespace="spotify")    # spotify_play, spotify_search, ...
# ... 11 more services
GitHub (11) Workspace (15) Memory (8) Gmail (4) Mastodon (9) Spotify (5) Discord (4) Telegram (4) Calendar (3) Sandbox (7) Scheduler (3) Web Scrape (3) RSS (4) YouTube (2) Web Search (1)

Adaptive Agent Loop

Mode A: Plan-Then-Execute

For multi-service orchestration tasks.

[
  {step: 1, tool: "web_search"},
  {step: 2, tool: "llm_generate",
   depends_on: 1},
  {step: 3, tool: "telegram_send",
   depends_on: 2}
]

Steps with depends_on receive outputs from referenced steps.

Mode B: Adaptive Loop

For coding tasks and iterative workflows.

while iterations < max:
  response = llm.chat(
    messages, tools, system
  )
  if response.has_tool_calls:
    result = call_tool(...)
    # LLM sees result next turn
  else:
    break  # done

50 iterations for code · 30 for general tasks

Context-Aware Tool Filtering

83 tools overwhelm small models. OpenPA dynamically filters by task context.

"Add a health check endpoint" → workspace github core = 34 tools
"Play some jazz" → media core = 15 tools
"Who am I?" → core = 8 tools
83 → 8

Enables 7–9B local models (Gemma, Qwen, Llama) to function effectively

RAG-Based Personalisation

Not just preferences — a semantic understanding that grows over time.

How it works

Acquire — LLM calls remember_about_user proactively
Embed — nomic-embed-text via Ollama (local, free)
Store — ChromaDB with user ID + category metadata
Retrieve — Top-10 by cosine similarity per message
Inject — Appended to system prompt with relevance scores

Example

User (day 1): "I'm a software dev
who loves hiking and Italian food."

→ Saves 2 memories (work, interests)

User (day 7): "Who am I?"

→ RAG retrieves:
  [work] Software developer (0.87)
  [interests] Hiking, Italian food (0.82)

→ Agent recalls everything

Vibe-Coding Workspace

The agent autonomously clones, reads, edits, tests, fixes, and ships code.

15 Workspace Tools

Explore: list_files, read_file, grep, find, inspect (AST)

Modify: write_file, edit_file, delete_file, install

Verify: run (pytest, npm build), check_syntax

Git: diff, commit_push, create, cleanup

4-Phase Adaptive Workflow

Understand — Clone, explore, grep patterns
Research — Web search for unfamiliar APIs
Implement — Edit code, check syntax
Verify (loop) — Run tests; if fail → fix → retest

Example: Vibe-Coding in Action

User: "Add a health check endpoint to the REST API."

ws_workspace_create — Clone repo, branch feature/health-check
list_files + read_file + grep — Explore codebase, learn conventions
edit_file — Add /api/health endpoint
check_syntax — py_compile + ruff ✓
run — pytest tests/ -v → 32/32 pass ✓   npm run build → ✓
commit_push + github_create_pr — Ship it!

Fully autonomous · if tests fail, it reads the error, fixes, and re-runs

Self-Evolution Safety

Not as scary as it sounds. 🛡️

Pull Requests, Not Merges

Changes are submitted as PRs. A human reviews before deployment.

Full Test Suite Must Pass

Broken code is never shipped. CI validates every push.

Isolated Workspace

Agent works on a cloned copy, not the live deployment.

Auto-Cleanup

Workspace is deleted after push. No resource leakage.

Limitations

  • SQLite limits horizontal scaling — needs PostgreSQL for production
  • Local models (7–9B) still struggle with complex multi-step tool-calling
  • No containerisation yet — Docker/K8s configs are future work
  • Security — OAuth tokens stored without encryption at rest
  • Test coverage — core tests exist, but no full end-to-end agent loop tests
